## Goal

Display Google Scholar citation counts on `/publications/` when we have them, and fall back to OpenAlex (the current source) when we don't. The current page already shows OpenAlex counts under a "scholar"-labelled badge — visually misleading. After this change, the badge will *actually* be Scholar when data is available, OpenAlex otherwise.

Constraints:
- GitHub Pages build cannot scrape Scholar (datacenter IP → blocked instantly).
- Pipeline must not break the site if the scrape is stale or empty.
- Manual weekly run from Kevin's laptop is acceptable.

## Data file shape

`_data/scholar_counts.yml`:

```yaml
# Auto-generated by scripts/fetch_scholar_counts.py. Edit via the script, not by hand.
fetched_at: "2026-05-25T22:30:00Z"
source: "google_scholar (via scholarly)"
counts:
  "10.1287/isre.2022.1160":
    count: 95
    title_match: 0.98
  "10.1287/isre.2022.1130":
    count: 70
    title_match: 0.96
    ...
```

DOI key is normalized: `strip()`, `.lower()`, leading `https?://doi.org/` removed. Same normalization in JS so lookups don't drift. **Note**: MISQ DOIs use mixed case (`10.25300/MISQ/...`). DOI is case-insensitive per spec; we lowercase consistently on both sides (Python writer + JS reader) so the lookup chain is self-consistent. Document this assumption in both files.

Top-level `fetched_at` reflects the run that produced this file. Per-entry timestamps were considered and dropped — incremental updates already preserve previous values by reading existing YAML first.

## Scrape script logic (`scripts/fetch_scholar_counts.py`)

### Inputs
- `_data/publications.yml` (read-only)
- Existing `_data/scholar_counts.yml` if present (used as a starting point — we update rather than rebuild from scratch, so a transient block on one paper doesn't lose 38 good entries).

### Per-paper algorithm
1. Skip if paper has no DOI.
2. *(Previously skipped forthcoming papers. Removed — working-paper cites are exactly where Scholar beats OpenAlex, and the match-verification step below is strict enough to reject wrong matches via the DOI cross-check.)*
3. Build query: `title + " " + first_author_last` (free-text, no field operators — Scholar doesn't support them).
4. Take first hit from `scholarly.search_pubs(query)`.
5. **Verify match (all three must hold):**
   - **Title similarity**: lowercased, punctuation-stripped Levenshtein ratio.
     - For titles with ≥ 8 words: require **≥ 0.85**.
     - For titles with < 8 words: require **≥ 0.92** (short titles are vulnerable to false positives).
   - **Year**: within ±1 of `pub.year` (preprints can predate).
   - **DOI cross-check** (when available): if `pub['bib']['doi']` is present in Scholar's response, lowercase-compare to our DOI. **Hard mismatch → skip regardless of title score.**
   - If any check fails → log warning, skip (keep previous value).
6. Capture `num_citations`.
7. Sleep `random.uniform(15, 45)` seconds (jitter — Scholar blocks bursts).
8. Write `_data/scholar_counts.yml` atomically after **every** paper (tmpfile + `os.replace` — atomic on APFS, no `fsync` needed since git handles durability). Loss of process = at most current paper in-flight.

### CLI flags
- `--limit N` — only scrape first N papers (smoke-test mode).
- `--only DOI` — scrape one specific DOI.
- `--dry-run` — fetch but don't write.
- `--no-jitter` — for debugging (caution: invites blocking).

(Dropped `--rebuild`: `rm _data/scholar_counts.yml && python scripts/fetch_scholar_counts.py` achieves the same thing with one less code path.)

### Error handling
- `MaxTriesExceededException` (CAPTCHA hit) → stop the batch (further requests will all fail). Print clear message + how many succeeded.
- Network errors → retry once with backoff, then skip paper.
- Empty `search_pubs` results → skip paper, log.
- Title mismatch → log, skip.
- All "skip" outcomes preserve the previous value in the YAML, so successive runs can converge.

### Output to stdout
A small summary table:
```
[ok]   10.1287/isre.2022.1160        82 → 95   (match 0.98)
[skip] 10.1287/isre.2024.1190        (forthcoming)
[fail] 10.25300/MISQ/2022/16959      (no match)
...
Wrote 12 updated, 22 unchanged, 5 skipped, 0 failed.
```

### `assets/js/research.js`

Modify `fetchCitations()` (currently at line ~722). **Critical**: must delete Scholar-covered DOIs from `doiMap` before the OpenAlex pass, AND bump the cache key so old caches (containing pre-Scholar data for all 39 DOIs) don't overwrite the new Scholar values:

```js
async function fetchCitations() {
  if (citationsApplied) return;
  citationsApplied = true;

  // Bumped from 'nh_openalex_citations' so the first deploy invalidates any
  // pre-existing cached blob that contains DOIs now covered by Scholar.
  const CACHE_KEY = 'nh_openalex_citations_v2';
  const CACHE_TTL = 24 * 60 * 60 * 1000;

  const anchors = Array.from(document.querySelectorAll('.pub-cite[data-doi]'))
                       .filter(a => a.dataset.doi);
  if (!anchors.length) return;

  const doiMap = {};
  anchors.forEach(a => { doiMap[a.dataset.doi.toLowerCase()] = a; });

  // 1. Apply Scholar counts synchronously (build-time data, fresh per page load).
  //    DOIs are already lowercased on both sides — see scrape script for the
  //    normalization rationale (MISQ DOIs are mixed-case).
  const scholar = window.SCHOLAR_COUNTS || {};
  Object.entries(scholar).forEach(([doi, count]) => {
    if (doiMap[doi] != null && count != null) {
      const span = doiMap[doi].querySelector('.pub-cite-count');
      if (span) span.textContent = Number(count).toLocaleString();
      doiMap[doi].dataset.citeSource = 'scholar';
      // Remove from doiMap so OpenAlex pass (and its cache) can't overwrite Scholar.
      delete doiMap[doi];
    }
  });

  // 2. Only ask OpenAlex for the residual DOIs.
  const dois = Object.keys(doiMap);
  if (!dois.length) return;

  // ... rest of existing OpenAlex flow, but using `dois` (the residual set)
  // and tagging anchors that get a value with data-cite-source="openalex".
}
```

- `data-cite-source="openalex"` → `title="OpenAlex (live)"`

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| Scholar blocks the laptop IP | Run weekly (low frequency), large jitter (15-45 s). If blocked, wait 24 h or use a different network. |
| `scholarly` package breaks | Pin version in requirements file. Bump deliberately. |
| Title-mismatch false positive (wrong paper's count attributed) | Title similarity threshold of 0.85 + year ±1 check. Errs on the side of skip. |
| Editorial / forthcoming papers have no Scholar entry | If Scholar returns nothing, `search_pubs` yields no first hit → `[fail]` log + skip. If Scholar returns a wrong paper, `verify_match()` rejects it via title + year + DOI cross-check → `[fail]` log + skip. Either way the badge falls back to OpenAlex → typically "—". |
| Data file goes stale | Top-level `fetched_at` is displayed in tooltip; operator notices visually. |
| Build fails because data file is missing | Liquid guards (`{% if site.data.scholar_counts %}`) — empty `SCHOLAR_COUNTS = {}` is valid JS. |

## Rollback

Worst case: `git rm _data/scholar_counts.yml`. JS sees empty/undefined `SCHOLAR_COUNTS`, OpenAlex flow runs for all DOIs. Site is back to current behaviour.

## Audit fixes (post-review)

Five concrete changes from the audit pass:

1. **Cache key bump**: `'nh_openalex_citations'` → `'nh_openalex_citations_v2'`. Without this, returning visitors get a cached blob containing OpenAlex counts for all 39 DOIs, and the OpenAlex `applyCountsToDOM` pass overwrites Scholar values that were applied synchronously a moment earlier. Fix is to also `delete doiMap[doi]` after each Scholar application so the OpenAlex pass operates on the strict residual.
2. **Title similarity threshold**: tiered — `< 8 words → ≥ 0.92`, `≥ 8 words → ≥ 0.85`. Short titles are vulnerable to false positives at the lenient threshold.
3. **Title normalization**: lowercase + strip punctuation before computing Levenshtein ratio. Without this, Chicago-cased YAML titles vs. Scholar's variable casing inflate the distance.
4. **DOI cross-check**: when Scholar's result includes a DOI in `pub['bib']['doi']`, hard-compare to ours. Mismatch → skip regardless of title score. (Belt-and-suspenders for the wrong-paper failure mode.)
5. **Dropped over-engineering**: per-entry `fetched_at`, per-entry `scholar_id`, `--rebuild` flag. None of these earn their complexity on a 39-paper personal site.

Verified-good claims that stood up to review:
- `DOMContentLoaded` init order: `renderPubs` is synchronous, so `fetchCitations` sees a complete DOM. No race.
- `os.replace` on APFS is atomic; no `fsync` needed.
- Liquid `{% if site.data.scholar_counts and site.data.scholar_counts.counts %}` correctly handles the absent-file rollback.
- DOI lowercase normalization is consistent on both sides of the lookup chain.
